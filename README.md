# X Bookmark Organizer

X Bookmark Organizer 是一个用于整理 X (Twitter) 书签的 Chrome 浏览器插件。它可以同步、筛选、分类和可视化你的 X 收藏内容，也可以记录本地浏览历史，并支持接入 OpenAI 兼容接口，让 AI 帮你生成分类建议。

这个项目的目标很简单：把 X 上散落的收藏内容变成一个可以搜索、筛选、统计和管理的本地知识库。

## 主要功能

- 同步 X 书签：在已登录 X 的浏览器环境中同步书签内容。
- 书签分类管理：支持新建分类、编辑分类、批量归类、忽略和删除本地记录。
- 时间筛选：支持按推文发布时间或本地入库时间筛选。
- 统计可视化：支持按日、周、月、半年、年查看收藏数量，并提供热力图和趋势折线图。
- 推文内容展示：保留正文、作者、链接、图片、视频等信息，方便回看。
- 关注博主管理：整理关注博主信息，并展示粉丝数、推文数、收藏关联等基础数据。
- 浏览记录保存：可选记录 X 浏览历史，数据保存在本地。
- AI 分类建议：支持自定义 API Base URL、API Key、模型名和分类提示词，让 AI 给出分类草稿。
- 数据导入导出：支持 JSON 备份，方便迁移和恢复。
- 本地优先：书签、分类、浏览记录和设置默认保存在 `chrome.storage.local`。

## 安装方式

项目会在 `output` 目录下生成两种交付物：

- `output/x-bookmark-organizer-0.6.12.crx`：打包后的 CRX 文件。
- `output/x-bookmark-organizer-0.6.12/`：解压后的插件文件夹，推荐用这个安装。

现代 Chrome/Edge 对直接安装 `.crx` 有限制，所以日常使用建议安装解压文件夹：

1. 打开 Chrome 或 Edge。
2. 地址栏输入 `chrome://extensions/`。
3. 打开右上角「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择 `E:\BaiduNetdiskDownload\X_bookmark\output\x-bookmark-organizer-0.6.12`。
6. 安装完成后，浏览器工具栏会出现插件图标。

如果只有 `.crx` 文件，也可以把后缀改成 `.rar` 或 `.zip` 后解压，再用「加载已解压的扩展程序」安装得到的文件夹。

## 使用方法

### 1. 同步书签

1. 先在浏览器里登录 `https://x.com/`。
2. 点击插件图标打开弹窗。
3. 点击「完整同步全部书签」。
4. 同步完成后，进入插件管理页查看书签。

如果 X 接口临时变化，插件也提供「扫描当前可见页」作为回退方式，但全量同步仍建议优先使用完整同步。

### 2. 整理分类

进入插件管理页后，可以在左侧分类区管理文件夹：

- 点击「+ 分类」新建分类。
- 点击分类查看该分类下的书签。
- 对未分类书签进行手动归类。
- 勾选多条书签后进行批量归类、忽略或删除。
- 给分类添加描述，方便 AI 更准确地判断。

### 3. 时间筛选和统计

管理页顶部提供时间筛选区：

- 时间依据：可选择「推文发布时间」或「收藏记录时间」。
- 时间范围：支持全部记录、按年、按半年、按月、按日和自定义范围。
- 展示粒度：支持自动、按年、按半年、按月、按周、按日。

页面会根据当前时间筛选展示：

- 左侧热力图：展示不同时间段内的收藏数量。
- 右侧折线图：展示收藏数量变化趋势。
- 统计卡片：展示累计收藏、峰值、活跃天数等信息。

点击热力图色块或统计行，可以直接筛出对应时间段的推文。

### 4. 查看收藏推文

筛选出某一天、某一周或某一个月后，下方会显示对应的收藏推文列表。推文会尽量保留原始结构，包括：

- 作者头像和昵称
- 推文正文
- 图片、视频和链接
- 发布时间
- 分类状态
- 本地笔记或操作按钮

为了避免收藏数量过多导致页面卡顿，插件不会一次性渲染全部记录，而是按时间段和分页逐步展示。

### 5. 关注博主管理

插件可以整理你关注或书签中出现的博主信息。管理页左侧可以查看博主列表，右侧可以继续查看书签统计和收藏推文。

博主信息会尽量展示：

- 昵称和用户名
- 粉丝数
- 推文数
- 关联收藏数量
- 基础热度数据

部分数据依赖 X 页面或接口能否提供，如果 X 当前页面没有暴露对应字段，插件会保留已有信息，不会伪造数据。

## AI 分类设置

插件支持 OpenAI Chat Completions 兼容接口。进入「AI 设置」后填写：

- API Base URL
- API Key
- 模型名
- 自定义分类提示词

示例：

```text
API Base URL: https://api.openai.com/v1
Model: gpt-4o-mini
```

也可以填写兼容接口的完整地址，例如：

```text
https://open.bigmodel.cn/api/coding/paas/v4/chat/completions
```

或者填写上级 base url：

```text
https://open.bigmodel.cn/api/coding/paas/v4
```

插件会自动识别是否已经包含 `/chat/completions`，避免重复拼接。

AI 的工作流程：

1. 在 AI 设置里填写接口信息。
2. 点击「测试连通」确认接口可用。
3. 点击「AI 整理未分类」。
4. AI 返回分类建议。
5. 你逐条确认、修改或拒绝。
6. 点击应用后才会真正写入分类。

AI 只会读取用于分类的书签文本、作者和已有分类信息，不会读取你的密码，也不会主动上传本地浏览记录。

## 数据与隐私

- 所有书签、分类、浏览记录、AI 设置默认保存在浏览器本地。
- 插件不会保存你的 X 密码。
- 插件不会主动把数据上传到第三方服务器。
- 如果你启用 AI 分类，只有参与分类的文本会发送到你填写的 AI 接口。
- API Key 存在本地浏览器存储中，请不要把截图或配置文件公开发布。
- 卸载插件会清除插件本地数据，建议重要数据先导出 JSON 备份。

## 本地开发

项目是原生 Chrome MV3 插件，主要使用 HTML、CSS 和 vanilla JavaScript，不依赖前端框架。

项目结构：

```text
X_bookmark/
├─ manifest.json
├─ background.js
├─ content.js
├─ popup/
│  ├─ popup.html
│  ├─ popup.css
│  └─ popup.js
├─ options/
│  ├─ options.html
│  ├─ options.css
│  └─ options.js
├─ lib/
│  ├─ util.js
│  ├─ storage.js
│  ├─ scraper.js
│  ├─ xapi.js
│  └─ ai.js
├─ icons/
├─ output/
└─ _pack.cjs
```

常用命令：

```bash
node --check lib/ai.js
node --check background.js
node _pack.cjs
```

打包后会在 `output` 目录生成新的 `.crx` 文件、`.zip` 文件和解压文件夹。

## 已知限制

- X 的内部接口可能随时变化，导致全量同步需要更新适配。
- X 不稳定提供真实「收藏时间」，所以默认更多使用「推文发布时间」做历史分布。
- AI 分类质量取决于模型能力、提示词和已有分类描述。
- 粉丝数、推文数、浏览量等博主数据依赖 X 页面可见信息，无法保证所有账号都完整。
- 大量书签场景下，插件会优先使用分段加载和统计视图，避免一次性渲染导致卡顿。
